Repo **is** larch Claude Code plugin. Edit here = ship to user. See `README.md` for install, features, env vars, skill catalog.

## Repository layout

Plugin ship whole repo. **Runtime surface**: `skills/`, `agents/`, `hooks/`, `scripts/`, `.claude-plugin/`. Rest supplementary (docs, CI config, `.claude/skills/`, dev settings).

## Common editing tasks

- **Changing a skill** → start at `skills/<name>/SKILL.md`, then trace every helper in `skills/<name>/scripts/`, `scripts/`, `skills/shared/`. Behavior split between prompt and scripts.
- **Adding/modifying the Code Reviewer archetype** → edit `skills/shared/reviewer-templates.md` (canonical), then run `bash scripts/generate-code-reviewer-agent.sh` to regen `agents/code-reviewer.md`. For any other reviewer archetype, follow general rule: identify canonical source and mirror updates to any generated outputs.
- **Changing a shared script** → edit `scripts/<name>.sh`, then grep for callers across `skills/`, `hooks/`, `.claude/settings.json`, `.github/workflows/`, other scripts.
- **Changing dev-only skills** → edit under `.claude/skills/bump-version/` or `.claude/skills/relevant-checks/`.
- **Docs or scripts only** → classified as PATCH.
- **Dialectic smoke test** → `scripts/dialectic-smoke-test.sh` = offline fixture-driven regression guard for `/design` Step 2a.5. Fixtures live under `tests/fixtures/dialectic/` (non-runtime path — everything outside named runtime surface supplementary). Run local with `bash scripts/dialectic-smoke-test.sh` or via `make smoke-dialectic`; CI run it in `smoke-dialectic` job. When changing `skills/shared/dialectic-protocol.md` Parser tolerance or Threshold Rules sections, update smoke test and/or fixtures in same PR.
