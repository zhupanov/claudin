# scripts/test-references-headers.sh — contract

`scripts/test-references-headers.sh` is the cross-skill structural regression guard for the progressive-disclosure **reference header triplet** (closes #308) and the **Contract-field line-range rejection** (closes #322). It scans every `skills/*/references/*.md` via flat glob and asserts (a) each file contains the three anchored headers `**Consumer**:`, `**Contract**:`, `**When to load**:` at line start, and (b) the Contract paragraph carries no `L` + digits + (ASCII hyphen `-`, en-dash `–`, or em-dash `—`) + digits line-range citation.

**Scope and scan rule**. The glob is flat: `skills/*/references/*.md`. Nested paths such as `skills/<skill>/references/<subdir>/*.md` are NOT scanned. If a future skill introduces a `references/` subtree, the harness will not see those files; a follow-up PR must either flatten the layout or widen the glob (e.g., `globstar` / `find`) and document the change here.

**Header match is anchored**. Each header is matched via `grep -E` against `^\*\*Consumer\*\*:`, `^\*\*Contract\*\*:`, `^\*\*When to load\*\*:`. Anchoring at line-start avoids false-positives when a body paragraph or a code-fenced example mentions one of the triplet tokens without actually declaring it as a section header. This is a deliberate tightening compared to the legacy `grep -Fq` whole-file substring check in the retired `scripts/test-implement-structure.sh` assertion (8).

**Contract-field line-range rejection**. For each `skills/*/references/*.md`, the harness extracts the Contract paragraph and rejects any citation matching the ERE pattern `(^|[^[:alnum:]_])L[0-9]+(-|–|—)[0-9]+`. The leading `(^|[^[:alnum:]_])` is a word-boundary guard — requiring a non-alphanumeric (and non-underscore) character before `L`, or the start of the line — so identifier-like substrings such as `modelL5-10` or `testLevelL100-200` don't false-positive on substring match. ASCII hyphen (`-`), en-dash (`–`), and em-dash (`—`) are all matched because GitHub Markdown rendering and author keyboards vary. **Paragraph extraction** starts at the anchored `^\*\*Contract\*\*:` header and stops at whichever comes first: (a) the next whitespace-only line (`^[[:space:]]*$`, tolerating trailing spaces / tabs) or (b) the next anchored `**Consumer**:` / `**When to load**:` triplet header — (b) is the guard against a missing trailing blank line causing awk to over-scan into the next section. The terminator is narrowed to the canonical triplet on purpose: a broader `^\*\*Word\*\*:` match could halt on an in-Contract callout such as a hypothetical `**Note**:` and miss a stale citation that follows it (false negative). **Alternation, not a character class**: `(-|–|—)` matches the en-dash and em-dash as byte-atomic multi-byte alternatives, so the check stays effective under `LC_ALL=C` where a character class `[-–—]` would parse each multi-byte character's UTF-8 bytes as separate class members and silently miss real matches. **Why this assertion exists**: the four `skills/implement/references/*.md` files historically carried stale source-of-truth line-range citations (`L522-528`, `L758-898`, `L1030-1123`, `L580-681`) that drifted silently as SKILL.md was edited later; v5.2.7 replaced all four with range-free descriptions (FINDING_4), and issue #322 was the OOS follow-up tracking the need for a regression guard so no future reference file can reintroduce the drift pattern. **Edit-sync rule**: when describing a Contract, cite the authoritative concept (e.g., "Phase 1 classification", "Step 2.4 collection", "the `--branch-info` 4-key requirement") rather than source-line ranges; if a future maintainer genuinely needs to cite a fixed range, update this harness to exclude that file and document the exemption. Separately, avoid line-start `**Word**:` bold-colon lines inside a Contract paragraph body unless separated by a blank line — prefer `**Word:**` (colon inside the bold) to avoid ambiguity with the triplet terminator. **Single-paragraph invariant**: the Contract block is a single paragraph. Do NOT introduce internal blank lines within the Contract body — the first whitespace-only line ends extraction, so any second Contract paragraph would be unscanned and could silently hide a stale line-range citation.

**Fail-closed on empty glob**. If no `skills/*/references/*.md` files are found, the harness fails rather than silently passing. The guard protects against a layout refactor that renames or moves `references/` directories without updating this harness's scope.

**Ownership split with sibling harnesses**:
- `scripts/test-implement-structure.sh` owns `/implement`-specific topology (top-level headings, MANDATORY ↔ reference binding, CI-parity focus-area enum, the `see Step N below|above` ban). As of #308, it no longer owns the Consumer/Contract/When-to-load triplet; that invariant moved here.
- `scripts/test-research-structure.sh` retains a stricter `/research`-local check that the triplet appears **in the first 20 lines** (opens-with), layered on top of the global presence check this harness enforces. Do not remove the `/research` first-20-lines check in favor of the global presence check — it is a tightening, not a duplicate.

**Wiring**:
- Invoked via `make test-references-headers` target in the root `Makefile`; included in the `test-harnesses` aggregate so `make lint` and CI's `agent-sync`/`test-harnesses` jobs pick it up transitively.
- Added to `agent-lint.toml`'s exclude list (same Makefile-only-reference pattern as other `test-*.sh` harnesses): agent-lint's dead-script rule does not follow Makefile-only references.

**Edit synchronization**:
- Any edit to `skills/*/references/*.md` that would remove or rename one of the three anchored headers must also update that file's header block so the harness continues to pass.
- Adding a new `references/` directory under a new skill (e.g., `skills/<new>/references/foo.md`) requires the triplet headers on all included `.md` files from day one.
- Moving to a nested layout (`references/<subdir>/*.md`) requires widening this harness's scan rule (glob or `find`-based) and updating this contract section.
- Do NOT reintroduce line-range citations inside a Contract paragraph — specifically `L` + digits + (any of ASCII hyphen `-`, en-dash `–`, or em-dash `—`) + digits. Use range-free conceptual descriptions. If a bona-fide exemption is needed, add the exempt path to the harness's Contract-field scan rule and document the reason here.
