# scripts/test-implement-structure.sh — contract

`scripts/test-implement-structure.sh` is the structural regression guard for `skills/implement/SKILL.md` and `skills/implement/references/` topology (closes #234). Nine assertions: (1) exactly one `^## Load-Bearing Invariants$` heading in `skills/implement/SKILL.md`; (2) exactly one `^## NEVER List$` heading; (3) exactly one `^## Rebase Checkpoint Macro$` heading; (4) at least 4 `MANDATORY — READ ENTIRE FILE` occurrences in SKILL.md (floor, not ceiling) AND each of the four expected reference filenames (`bump-verification.md`, `conflict-resolution.md`, `pr-body-template.md`, `rebase-rebump-subprocedure.md`) appears on a `MANDATORY — READ ENTIRE FILE` line in SKILL.md (step-to-reference binding); (5) three byte-pinned verbosity-suppressed literals present in SKILL.md (`⏩ 1.m: design plan | update main — already at latest`, `⏩ 1.r: design plan | rebase — already pushed`, `⏩ 1.r: design plan | rebase — already at latest main`); (6) CI-parity focus-area enum check — at least one line in SKILL.md must contain the literal `code-quality / risk-integration / correctness / architecture`, AND **every** such line must also contain `security` on the same line (fails on any enum line missing `security`, mirroring `.github/workflows/ci.yaml` L121/L125 — the `agent-sync` job's `UNQUOTED_FILES` loop enforces per-line coverage, not just the presence of one qualifying line — which prevents a false-pass when the five tokens appear separately in unrelated prose while the actual Cursor/Codex quick-review prompt strings regress); (7) the four `skills/implement/references/*.md` files exist with expected names; (8) each reference file contains the `**Consumer**:`, `**Contract**:`, `**When to load**:` header triplet; (9) zero `see Step N below` / `see Step N above` patterns in any `references/*.md` (progressive-disclosure invariant — references must not back-reference parent SKILL.md step numbers with direction words; the pattern is narrow, requiring both a step number AND a direction word, matched case-insensitively so sentence-initial `See Step 8 below` variants are also caught; legitimate `see Step 8` cross-refs without direction words are permitted). Ownership split with the companion harness `scripts/test-implement-rebase-macro.sh`: the macro harness owns Rebase Checkpoint Macro placement and registry mechanics; this harness owns top-level section headings, the MANDATORY ↔ reference-file binding, the CI-parity focus-area enum check, reference-file contract headers, and the no-`see Step N below|above` invariant in references/*.md. Intentional overlap: assertion (3) (single `## Rebase Checkpoint Macro` heading) duplicates the macro harness's assertion (A), and assertion (5) (verbosity literals) duplicates the macro harness's assertion (D) — accepted duplication (each harness stays one idea). Wired into `make lint` via the `test-implement-structure` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to `skills/implement/SKILL.md` (any of the three top-level headings, the MANDATORY pointers, the verbosity literals, the focus-area enum in the Cursor/Codex quick-review prompts) or to any `skills/implement/references/*.md` (renaming, removing, altering the Consumer/Contract/When-to-load triplet, or introducing `see Step N below|above` prose) must stay in sync with this harness's literal assertions.
