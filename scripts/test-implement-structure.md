# scripts/test-implement-structure.sh — contract

`scripts/test-implement-structure.sh` is the structural regression guard for `skills/implement/SKILL.md` and `skills/implement/references/` topology (closes #234). **Twelve assertions**: (1) exactly one `^## Load-Bearing Invariants$` heading in `skills/implement/SKILL.md`; (2) exactly one `^## NEVER List$` heading; (3) exactly one `^## Rebase Checkpoint Macro$` heading; (4) at least 5 `MANDATORY — READ ENTIRE FILE` occurrences in SKILL.md (floor, not ceiling) AND each of the five expected reference filenames (`anchor-comment-template.md`, `bump-verification.md`, `conflict-resolution.md`, `pr-body-template.md`, `rebase-rebump-subprocedure.md`) appears on a `MANDATORY — READ ENTIRE FILE` line in SKILL.md (step-to-reference binding); (5) three byte-pinned verbosity-suppressed literals present in SKILL.md (`⏩ 1.m: design plan | update main — already at latest`, `⏩ 1.r: design plan | rebase — already pushed`, `⏩ 1.r: design plan | rebase — already at latest main`); (6) CI-parity focus-area enum check — at least one line in SKILL.md must contain the literal `code-quality / risk-integration / correctness / architecture`, AND **every** such line must also contain `security` on the same line (fails on any enum line missing `security`, mirroring `.github/workflows/ci.yaml` L121/L125 — the `agent-sync` job's `UNQUOTED_FILES` loop enforces per-line coverage, not just the presence of one qualifying line — which prevents a false-pass when the five tokens appear separately in unrelated prose while the actual Cursor/Codex quick-review prompt strings regress); (7) the five `skills/implement/references/*.md` files exist with expected names; (8) zero `see Step N below` / `see Step N above` patterns in any `references/*.md` (progressive-disclosure invariant — references must not back-reference parent SKILL.md step numbers with direction words; the pattern is narrow, requiring both a step number AND a direction word, matched case-insensitively so sentence-initial `See Step 8 below` variants are also caught; legitimate `see Step 8` cross-refs without direction words are permitted); (9) load-bearing marker literals in `skills/implement/references/anchor-comment-template.md` (closes #323; migrated from `pr-body-template.md` per umbrella #348 Phase 3) — (9a) three byte-pinned marker literals must be present in `anchor-comment-template.md` (`Accepted OOS (GitHub issues filed)`, `| OOS issues filed |`, `<details><summary>Execution Issues</summary>`), which the Step 9a.1 OOS issue-filing pipeline parses and rewrites (the OOS placeholder and the Run Statistics OOS cell in the anchor comment's `oos-issues` + `run-statistics` sections) and the Step 11 post-execution anchor refresh parses and rewrites (the Execution Issues details block in the anchor's `execution-issues` section) — fixed-string matching because the literals contain regex metachars (`|`, `<`, `>`); (9b) SKILL.md must reference `anchor-comment-template.md` at least 3 times (Step 0.5 MANDATORY pointer + Step 9a.1 prose binding + Step 11 prose binding), guarding against a future edit that keeps the MANDATORY pointer (which assertion (4) already verifies) but orphans Step 9a.1 or Step 11 from the extracted reference — both steps delegate their procedure to `anchor-comment-template.md` sections; (9c) SKILL.md must reference `pr-body-template.md` at least 1 time — the Step 9a MANDATORY pointer — lower floor than pre-Phase-3 (was ≥3) since rich report content migrated to the anchor comment; (10) cross-skill bail-token pin (umbrella #348 Phase 4) — `skills/implement/SKILL.md` must contain the literal `IMPLEMENT_BAIL_REASON=adopted-issue-closed`, the exact token `/implement` Step 0.5 Branch 2 emits on stdout when the adopted tracking issue is CLOSED; `/fix-issue` Step 6a greps captured `/implement` output for this literal to branch to a specific warning + skip-to-cleanup path; the paired consumer-side assertion lives in `skills/fix-issue/scripts/test-fix-issue-bail-detection.sh`, so a rename of the token is a dual-repo change caught by CI; (11) Phase 5 (umbrella #348) `rebase-rebump-subprocedure.md` reference set — (11a) references `anchor-comment-template.md` at least once (Contract citation after retargeting step 6 from PR-body refresh to anchor `version-bump-reasoning` refresh), (11b) references `tracking-issue-read.sh --sentinel` at least once (step 6a sentinel read), (11c) references `assemble-anchor.sh` and `upsert-anchor` at least once each (step 6d assemble + step 6e upsert), (11d) zero invocation lines of `${CLAUDE_PLUGIN_ROOT}/scripts/gh-pr-body-read.sh` or `${CLAUDE_PLUGIN_ROOT}/scripts/gh-pr-body-update.sh` — scoped to invocation patterns only, so historical/explanatory prose mentioning the old script names remains permitted; (12) Phase 5 single-source-of-truth invariant for `SECTION_MARKERS` — `scripts/tracking-issue-write.sh` must reference `anchor-section-markers.sh` at least once (positive source-call check) and must NOT contain a standalone `SECTION_MARKERS=(` declaration (negative — the old inline declaration was removed; any re-inline would silently diverge `tracking-issue-write.sh`'s ordering from `assemble-anchor.sh`). **Cross-skill Consumer/Contract/When-to-load header-triplet ownership**: moved to `scripts/test-references-headers.sh` as of #308 — that harness scans every `skills/*/references/*.md` repo-wide. This harness no longer owns that invariant. Ownership split with the companion harness `scripts/test-implement-rebase-macro.sh`: the macro harness owns Rebase Checkpoint Macro placement and registry mechanics; this harness owns top-level section headings, the MANDATORY ↔ reference-file binding, the CI-parity focus-area enum check, the no-`see Step N below|above` invariant in references/*.md, and the anchor-comment-template.md load-bearing marker literals plus the `anchor-comment-template.md` and `pr-body-template.md` reference counts in SKILL.md. Intentional overlap: assertion (3) (single `## Rebase Checkpoint Macro` heading) duplicates the macro harness's assertion (A), and assertion (5) (verbosity literals) duplicates the macro harness's assertion (D) — accepted duplication (each harness stays one idea). Wired into `make lint` via the `test-implement-structure` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to `skills/implement/SKILL.md` (any of the three top-level headings, the MANDATORY pointers, the verbosity literals, the focus-area enum in the Cursor/Codex quick-review prompts, the `anchor-comment-template.md` references at Step 0.5 / 9a.1 / 11, the `pr-body-template.md` reference at Step 9a, the `IMPLEMENT_BAIL_REASON=adopted-issue-closed` bail token), to any `skills/implement/references/*.md` (renaming, removing, or introducing `see Step N below|above` prose; renaming or removing any of the three load-bearing marker literals in `anchor-comment-template.md`; swapping `rebase-rebump-subprocedure.md`'s Step 6 algorithm away from `tracking-issue-read.sh --sentinel` / `assemble-anchor.sh` / `upsert-anchor`; reintroducing a `gh-pr-body-read.sh` or `gh-pr-body-update.sh` invocation in the sub-procedure), or to `scripts/tracking-issue-write.sh` (re-inlining `SECTION_MARKERS=(` or removing the `anchor-section-markers.sh` source) must stay in sync with this harness's literal assertions. For the Consumer/Contract/When-to-load header triplet on any `skills/*/references/*.md`, see `scripts/test-references-headers.md`.
