# scripts/test-research-structure.sh — contract

`scripts/test-research-structure.sh` is the structural regression guard for the `/research` skill's 2-reference progressive-disclosure topology (`research-phase.md` owns Step 1 body; `validation-phase.md` owns Step 2 body). Six assertions: (1) `skills/research/SKILL.md` exists; (2) both `skills/research/references/research-phase.md` and `skills/research/references/validation-phase.md` exist; (3) each reference file is named on a `MANDATORY — READ ENTIRE FILE` line in `SKILL.md` AND that same line also carries the reciprocal `Do NOT load <other-reference>` guard (line-scoped by construction so a future edit cannot split the MANDATORY and the Do-NOT-load into different paragraphs without the harness catching the drift — prevents cross-phase context pollution when either reference is skipped); (4) each `references/*.md` **opens with** the `**Consumer**:` / `**Contract**:` / `**When to load**:` header triplet in the first 20 lines of the file (tightens the `/implement` precedent's whole-file grep in `test-implement-structure.sh` assertion 8 — matches this sibling contract's literal "opens with" promise, so a future edit that buries the triplet mid-file is caught); (5) `RESEARCH_PROMPT` identifier and the prompt-body opening substring `You are researching a codebase to answer this question` both appear in `research-phase.md` (byte-drift detection for the behavior-bearing literal); (6) reviewer XML wrapper tags `<reviewer_research_question>` and `<reviewer_research_findings>` appear in `validation-phase.md` (pin for the prompt-injection hardening delimiters). Wired into `make lint` via the `test-research-structure` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to `skills/research/SKILL.md` Step 1/Step 2 MANDATORY directives, to either reference file's header triplet, or to the pinned literal substrings must stay in sync with the harness's literal assertions.
