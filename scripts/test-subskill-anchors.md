# scripts/test-subskill-anchors.sh — contract

`scripts/test-subskill-anchors.sh` is the heading-anchor regression harness for `skills/shared/subskill-invocation.md` (closes #236). It scans **only** that one file for backticked `` `<path>/SKILL.md § <heading>` `` citations and asserts each resolves to a `## <heading>` or `### <heading>` line in the referenced SKILL.md (exact string match, trailing whitespace tolerated on the target line, both `##` and `###` headings accepted). Fail-closed on parse/IO errors; fenced code blocks (any opener of 3+ backticks, including the doc's quadruple-backtick banner examples) are skipped; a minimum-citation floor (`MIN_CITATIONS=10`) guards against extractor regressions silently validating zero or few anchors. This contract is orthogonal to `scripts/lint-skill-invocations.py`, which enforces sub-skill-invocation wording (Pattern A/B + `via the Skill tool`) inside SKILL.md bodies — the new harness checks that citation anchors still resolve after renames. Wired into `make lint` via the `test-subskill-anchors` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to citations in `skills/shared/subskill-invocation.md`, or to the cited `## <heading>` / `### <heading>` lines in any referenced SKILL.md, must stay in sync — the harness is the mechanical enforcer.
