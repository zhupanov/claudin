# scripts/parse-skill-judge-grade.sh — contract

`scripts/parse-skill-judge-grade.sh` is the shared `/skill-judge` output parser consumed by the `/loop-improve-skill` driver.sh (post-#273 single caller; both in-loop per-iter grade parse and post-iter-cap final re-evaluation) grade-gated termination contract. Reads one positional path argument (a `/skill-judge` output file) and emits KV stdout: `PARSE_STATUS=ok|missing_table|missing_file|bad_row|empty_file`, `GRADE_A=true|false`, `NON_A_DIMS=D2,D7` (when ok), `TOTAL_NUM`/`TOTAL_DEN` (when ok), per-dim `D1_NUM`/`D1_DEN` through `D8_NUM`/`D8_DEN` (when ok), and `OVERALL_GRADE=A|B|C|D|F` (when ok). **Fail-closed contract**: any non-ok `PARSE_STATUS` forces `GRADE_A=false` so the loop continues iterating rather than silently exiting as grade A on a parser failure or judge-output drift. **Defensive consistency rule scoped to ok parses only** (per FINDING_8): when `PARSE_STATUS=ok` AND `GRADE_A=false` AND `NON_A_DIMS` is empty, override to `PARSE_STATUS=bad_row`; non-ok statuses (e.g. `missing_table`, `empty_file`) preserve their original `PARSE_STATUS` unchanged because they legitimately have empty `NON_A_DIMS`. Per-dimension integer thresholds: `D1>=18/20`, `D2..D6+D8>=14/15`, `D7>=9/10` (equivalent to score/max >= 0.90 each — A in every dimension). Grammar pinned to `/skill-judge`'s SKILL.md "Step 5: Generate Report" template: locates FIRST `## Dimension Scores` heading; section MUST contain a pipe table with exactly 8 rows labeled `D1` through `D8` in that order, where the 2nd and 3rd pipe cells are integer score/max matching the expected per-dim max; anything else → `PARSE_STATUS=bad_row`. Single caller post-#273: `skills/loop-improve-skill/scripts/driver.sh` invokes it both in-loop per iteration (Phase 1.v grade parse + grade-A short-circuit + grade-history append) and at Step 5a on the iter-cap path (final post-iter-cap re-evaluation + close-out justification composition). Exit code 0 always (fail-closed contract is on stdout, not exit code); exit 1 only on argument errors. Bash 3.2 compatible (uses parallel index arrays + case-statement lookup helpers, NOT `declare -A` associative arrays). `scripts/test-parse-skill-judge-grade.sh` is its regression harness: 14 fixture cases under `mktemp -d` covering the full `PARSE_STATUS` taxonomy, per-dim threshold edges (case `(j)` exercises the `D1` max=20 arithmetic branch distinct from D2-D6+D8 max=15 and D7 max=10), the verbatim `/skill-judge` Step-5 report template (case `(k)` for upstream-grammar pinning per FINDING_4), and the FINDING_8 defensive-rule scoping (case `(l)` asserts `missing_table` is NOT rewritten to `bad_row`). Wired into `make lint` via the `test-parse-skill-judge-grade` target; added to `agent-lint.toml`'s exclude list (Makefile-only reference). Edits to the parser must stay in sync with the harness — and edits to the parser's grammar-pinning assumptions (heading text, column order, integer-cell rule) must be cross-referenced against the upstream `/skill-judge` SKILL.md's Step 5 template in the same PR.
