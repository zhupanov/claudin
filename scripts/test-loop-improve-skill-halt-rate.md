# scripts/test-loop-improve-skill-halt-rate.sh — contract

Also documents `scripts/lib-loop-improve-halt-ledger.sh` (sourced-only library), `scripts/test-lib-loop-improve-halt-ledger.sh` (library regression harness), and `tests/fixtures/loop-halt-rate/SKILL.md` (deficient throwaway fixture).

`scripts/test-loop-improve-skill-halt-rate.sh` is the **opt-in** halt-rate regression probe for `/larch:loop-improve-skill` (closes #278 under the halt-problem umbrella #273). It invokes `claude -p "/larch:loop-improve-skill loop-halt-rate"` N times end-to-end against a throwaway fixture skill at `tests/fixtures/loop-halt-rate/SKILL.md`, measures how often the outer loop halts mid-turn after `/skill-judge` returns (the halt-of-interest from #273), and emits `HALT_RATE=<halted>/<total>` + `PROBE_STATUS=ok|skipped_no_claude|error` + per-status and per-location breakdowns on stdout. Primary classification is via log parsing (outer's stdout `EXIT_REASON=` marker and presence/absence thereof) rather than filesystem sentinels, because outer's Step 6 cleanup removes `LOOP_TMPDIR` before `claude -p` returns on happy-path exits — filesystem-sentinel forensics applies only to `halt_mid_turn` runs where the outer itself halted before reaching Step 6. Per-run isolation: `mktemp -d` scratch dir with `git init` + bare origin provisioned (`git init --bare` then `git push origin main`) so the outer's `session-setup.sh → preflight.sh → git fetch origin main` chain does not abort before emitting `SESSION_TMPDIR=`; PATH-shimmed `gh` stub (no live GitHub side effects); `timeout --kill-after=10` wrapper (GNU coreutils `timeout` or macOS `gtimeout`) for process-group kill on hang; `LARCH_ROOT` captured before any `cd` and passed via `claude --plugin-dir`. Exit non-zero on missing `claude` binary per issue #278 explicit contract. `/im` is expected to fail under the stubbed gh; the resulting `ITER_STATUS=im_verification_failed` is classified as `completed_by_outer` (NOT a halt — the halt-of-interest fires at `/skill-judge` return, well before `/im`). Referenced only from `Makefile` (`halt-rate-probe` target, explicitly NOT a `make lint` prerequisite — opt-in only) and `README.md`; added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. `scripts/lib-loop-improve-halt-ledger.sh` is the sourced-only shell library consumed by the probe; it defines `classify_halt_location <LOOP_TMPDIR>` which scans for `iter-<N>-done.sentinel` / `iter-<N>-<substep>.done` files, finds the highest observed iteration, and emits `ITER=<N>`, `LAST_COMPLETED=<token>`, and byte-exact `HALT_LOCATION_CLAUSE=<canonical clause>` — `clause_for_last_completed()` in `scripts/lib-loop-improve-halt-ledger.sh` is the canonical taxonomy source (any edit to the `case` block MUST be mirrored in `scripts/test-lib-loop-improve-halt-ledger.sh`'s literal-pin assertions in the same PR). No shebang; sourced only. Added to `agent-lint.toml`'s exclude list because agent-lint does not follow `source` directives. `scripts/test-lib-loop-improve-halt-ledger.sh` is its regression harness (13 cases under `mktemp -d` covering empty dir, nonexistent dir, empty-string arg, each per-substep sentinel, multi-iter highest-iter scan, and empty-sentinel-treated-as-missing per `-s` semantics), wired into `make lint` via the `test-lib-halt-ledger` target; added to `agent-lint.toml`'s exclude list (Makefile-only reference). Edits to the ledger library must stay in sync with the harness. `tests/fixtures/loop-halt-rate/SKILL.md` is the throwaway fixture skill the probe copies into each per-run scratch repo at `skills/loop-halt-rate/SKILL.md`; it is deliberately deficient so `/skill-judge` consistently grades below A and the loop iterates — **do NOT expand into a real skill**; the fixture's minimality is load-bearing for reproducing the halt surface.
