# scripts/verify-skill-called.sh — contract

Also documents `scripts/lib-count-commits.sh` (sourced-only shared library) and `scripts/test-verify-skill-called.sh` (regression harness).

`scripts/verify-skill-called.sh` is a generic mechanical post-invocation verifier for `Skill` tool calls (closes #160). Three mutually-exclusive modes: `--sentinel-file <path>` (file exists, is a regular file, and is non-empty), `--stdout-line <regex> --stdout-file <path>` (captured stdout contains a line matching the POSIX ERE, matched via `LC_ALL=C grep -E -q -- …` to pin locale and block option injection; empty regex rejected as argument error; grep exit 2 is treated as an internal fault per the fail-closed contract), and `--commit-delta <expected> --before-count <N>` (commit count ahead of main increased by exactly the expected amount). Emits `VERIFIED=true|false` and `REASON=<token>` on stdout; exit 0 for pass/fail, exit 1 for argument errors and internal faults (fail-closed). Intended as a defense-in-depth gate for `Skill` calls whose child skills have no dedicated domain-specific verifier. Commit counting is **shared** with `scripts/check-bump-version.sh` via `scripts/lib-count-commits.sh`; `check-bump-version.sh` still owns the `--mode pre`/`--mode post` bump workflow (HAS_BUMP/VERIFIED/STATUS stdout contract — see `scripts/check-bump-version.md`), while `verify-skill-called.sh` adds generic sentinel/stdout/commit-delta modes. In `/implement` Step 8, the sentinel-file check runs alongside `check-bump-version.sh --mode post` — the two are complementary (sentinel = "skill wrote its artifact"; post = "skill created a commit"). `scripts/lib-count-commits.sh` is a sourced-only shell library (no shebang, not invokable directly) that defines `count_commits` for both `scripts/check-bump-version.sh` and `scripts/verify-skill-called.sh`. It distinguishes `ok` / `missing_main_ref` / `git_error` via a file-based side channel (`COUNT_COMMITS_STATUS_FILE`), preserves the existing `WARN: check-bump-version.sh:` stderr prefix for log parity, and explicitly documents that `.claude/skills/bump-version/scripts/classify-bump.sh`'s merge-base logic is a different concept intentionally not migrated. Do NOT invoke `lib-count-commits.sh` directly — it is sourced by the two consumers above. `scripts/test-verify-skill-called.sh` is the regression harness covering all three modes plus the cwd-neutral source chain via `check-bump-version.sh`, wired into `make lint` via the `test-verify-skill-called` target. Both `scripts/lib-count-commits.sh` and `scripts/test-verify-skill-called.sh` are added to `agent-lint.toml`'s exclude list because agent-lint does not follow `source` directives or Makefile-only references.
