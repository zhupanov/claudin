# scripts/test-review-structure.sh ‚Äî contract

`scripts/test-review-structure.sh` is the structural regression guard for the `/review` skill's progressive-disclosure topology (closes #306, hardened in #318). Nine assertions: (1) `skills/review/SKILL.md` and `skills/review/references/` exist; (2) baseline expected reference files `skills/review/references/domain-rules.md` and `skills/review/references/voting.md` exist; (3) every `skills/review/references/*.md` file on disk is named on at least one `MANDATORY ‚Äî READ ENTIRE FILE` line in `SKILL.md` using the `references/<basename>` path-token form followed by end-of-line or a non-filename character outside `[A-Za-z0-9._-]` (bidirectional orphan detection ‚Äî filesystem enumeration is authoritative so a newly added reference that lacks a MANDATORY pointer fails closed; the path-token boundary prevents both a name-containing-name false-pass like `my-voting.md` satisfying the check for `voting.md` AND a suffix/extension false-pass like `foo.md.bak` satisfying the check for `foo.md`); (4) each baseline expected reference (`domain-rules.md`, `voting.md`) appears on at least one MANDATORY line ‚Äî logically implied by (3) once the filesystem matches the baseline, but kept as a distinct check for clearer diagnostics when the baseline pair specifically regresses; (5) line-scoped callsite pins (#318, pattern parallel to `test-research-structure.sh`'s reciprocal Do-NOT-load pins) ‚Äî (5a) a single `SKILL.md` line carries `MANDATORY ‚Äî READ ENTIRE FILE`, `Step 3` (with a non-word-char boundary so `Step 3a` / `Step 30` / `Step 3f` do NOT false-pass), and `references/domain-rules.md` together (pinning `domain-rules.md` to the Step 3 entry callsite); (5b) a single line carries `MANDATORY ‚Äî READ ENTIRE FILE`, `round 1` (case-insensitive ‚Äî matches `In round 1` too ‚Äî with a non-word-char boundary so `round 10` / `round 11` do NOT false-pass), and `references/voting.md` together (pinning `voting.md` to the round-1 branch callsite); (5c) a single line carries `Do NOT load` and `references/voting.md` together (reciprocal rounds-2+ guard). Line-scoped by construction via a grep pipeline that threads each token through its own filter stage while preserving line granularity; a future edit that splits a directive across lines (e.g., moving `voting.md`'s MANDATORY to Step 3 entry or dropping the Do-NOT-load guard) fails closed. Reference-path boundary character outside `[A-Za-z0-9._-]` mirrors (3)/(4); (6) CI-parity focus-area enum check ‚Äî at least one line in `SKILL.md` contains the literal `code-quality / risk-integration / correctness / architecture` AND every such line also contains `security` on the same line (fails on any enum line missing `security`, mirroring the `agent-sync` job's `UNQUOTED_FILES` loop in `.github/workflows/ci.yaml` ‚Äî the loop referenced by name rather than line number so the harness does not rot when the workflow changes; per-line enforcement prevents a false-pass when the five tokens appear separately in unrelated prose while the Cursor/Codex quick-review prompt strings regress; same approach as `test-implement-structure.sh` assertion 6); (7) `SKILL.md` contains the anti-halt banner substring `**Anti-halt continuation reminder.**`; (8) `SKILL.md` contains the micro-reminder substring `Continue after child returns` (narrow token without bold wrappers or trailing period, matching `scripts/test-anti-halt-banners.sh`'s `MICRO_SIGNATURE` so a future loop-internal variant like `**Continue after child returns (loop-internal).**` still matches ‚Äî the canonical contract in `skills/shared/subskill-invocation.md` is this narrower token); (9) each `skills/review/references/*.md` opens with `**Consumer**:` AND `**Binding convention**:` in the first 20 lines (the `/review` native 2-line header schema ‚Äî deliberately NOT the `/implement` `Consumer`/`Contract`/`When to load` triplet because `/review` references use a different header schema; `head -n 20` gate matches `test-research-structure.sh` and `test-design-structure.sh`, so a future edit that buries the headers mid-file fails closed). Intentional overlap: assertions (7) and (8) duplicate per-skill coverage already provided by `scripts/test-anti-halt-banners.sh` (whose `ORCHESTRATORS` array includes `skills/review/SKILL.md` and whose `BANNER_SIGNATURE` / `MICRO_SIGNATURE` pin the same substrings). The duplication is intentional for single-file fail locality ‚Äî accepted duplication, matching the `test-implement-structure.sh` precedent which similarly overlaps with `test-implement-rebase-macro.sh`. Wired into `make lint` via the `test-review-structure` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to `skills/review/SKILL.md` (MANDATORY pointers including the Step-3-entry and round-1-branch callsites, the `Do NOT load` rounds-2+ guard for `voting.md`, the Cursor/Codex quick-review prompt lines carrying the focus-area enum, the anti-halt banner text, the `Continue after child returns` micro-reminder text) or to any `skills/review/references/*.md` (renaming, removing, altering the opening `**Consumer**:` / `**Binding convention**:` header lines, or adding a new reference without a MANDATORY pointer in `SKILL.md`) must stay in sync with this harness's literal assertions.
