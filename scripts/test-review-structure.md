# scripts/test-review-structure.sh — contract

`scripts/test-review-structure.sh` is the structural regression guard for the `/review` skill's progressive-disclosure topology (closes #306, hardened in #318; two-mode activation pins replace former slice-activation pins; substantive-validation flag pin added per #661; description-mode dual-list-contract pins added per #659). Eighteen assertions: (1) `skills/review/SKILL.md` and `skills/review/references/` exist; (2) baseline expected reference files `skills/review/references/domain-rules.md` and `skills/review/references/voting.md` exist; (3) every `skills/review/references/*.md` file on disk is named on at least one `MANDATORY — READ ENTIRE FILE` line in `SKILL.md` using the `references/<basename>` path-token form followed by end-of-line or a non-filename character outside `[A-Za-z0-9._-]` (bidirectional orphan detection — filesystem enumeration is authoritative so a newly added reference that lacks a MANDATORY pointer fails closed; the path-token boundary prevents both a name-containing-name false-pass like `my-voting.md` satisfying the check for `voting.md` AND a suffix/extension false-pass like `foo.md.bak` satisfying the check for `foo.md`); (4) each baseline expected reference (`domain-rules.md`, `voting.md`) appears on at least one MANDATORY line — logically implied by (3) once the filesystem matches the baseline, but kept as a distinct check for clearer diagnostics when the baseline pair specifically regresses; (5) line-scoped callsite pins (#318, pattern parallel to `test-research-structure.sh`'s reciprocal Do-NOT-load pins) — (5a) a single `SKILL.md` line carries `MANDATORY — READ ENTIRE FILE`, `Step 3` (with a non-word-char boundary so `Step 3a` / `Step 30` / `Step 3f` do NOT false-pass), and `references/domain-rules.md` together (pinning `domain-rules.md` to the Step 3 entry callsite); (5b) a single line carries `MANDATORY — READ ENTIRE FILE`, `round 1` (case-insensitive — matches `In round 1` too — with a non-word-char boundary so `round 10` / `round 11` do NOT false-pass), and `references/voting.md` together (pinning `voting.md` to the round-1 branch callsite); (5c) a single line carries `Do NOT load` and `references/voting.md` together (reciprocal rounds-2+ guard). Line-scoped by construction via a grep pipeline that threads each token through its own filter stage while preserving line granularity; a future edit that splits a directive across lines (e.g., moving `voting.md`'s MANDATORY to Step 3 entry or dropping the Do-NOT-load guard) fails closed. Reference-path boundary character outside `[A-Za-z0-9._-]` mirrors (3)/(4); (6) CI-parity focus-area enum check — at least one line in `SKILL.md` contains the literal `code-quality / risk-integration / correctness / architecture` AND every such line also contains `security` on the same line (fails on any enum line missing `security`, mirroring the `agent-sync` job's `UNQUOTED_FILES` loop in `.github/workflows/ci.yaml` — the loop referenced by name rather than line number so the harness does not rot when the workflow changes; per-line enforcement prevents a false-pass when the five tokens appear separately in unrelated prose while the Cursor/Codex quick-review prompt strings regress; same approach as `test-implement-structure.sh` assertion 6); (7) `SKILL.md` contains the anti-halt banner substring `**Anti-halt continuation reminder.**`; (8) `SKILL.md` contains the micro-reminder substring `Continue after child returns` (narrow token without bold wrappers or trailing period, matching `scripts/test-anti-halt-banners.sh`'s `MICRO_SIGNATURE` so a future loop-internal variant like `**Continue after child returns (loop-internal).**` still matches — the canonical contract in `skills/shared/subskill-invocation.md` is this narrower token); (9) each `skills/review/references/*.md` opens with `**Consumer**:` AND `**Binding convention**:` in the first 20 lines (the `/review` native 2-line header schema — deliberately NOT the `/implement` `Consumer`/`Contract`/`When to load` triplet because `/review` references use a different header schema; `head -n 20` gate matches `test-research-structure.sh` and `test-design-structure.sh`, so a future edit that buries the headers mid-file fails closed); (10) two-mode activation pin — at least one `SKILL.md` line carries `--diff` AND `positional description` (case-insensitive) together; pipeline-threaded so a future edit that splits the activation directive across lines fails closed; (11) verbatim --diff+description mutual-exclusion abort message pin — `SKILL.md` contains the literal string `**⚠ --diff cannot be combined with a description. Use --diff alone for branch diff review, or provide a description without --diff. Aborting.**` (`grep -Fq` so any wording drift fails closed); (12) verbatim no-args error abort message pin — `SKILL.md` contains the literal string `**⚠ /review requires either --diff (branch diff review) or a description of what to review.` (`grep -Fq` same rationale as (11)). Together (10)/(11)/(12) anchor the two-mode activation contract so a future edit cannot regress it silently; (13) substantive-validation flag pin (#661, pattern parallel to (10)) — at least one `SKILL.md` line carries `collect-agent-results.sh`, `--timeout 1860`, `--substantive-validation`, AND `--validation-mode` together; line-scoped via grep pipeline so a future edit that splits the Step 3a collector invocation across lines or drops either flag fails closed under `set -o pipefail`. Anchors the issue #661 contract that banner-only reviewer output (e.g., `Authentication required`) is rejected as `STATUS=NOT_SUBSTANTIVE` rather than passing as `STATUS=OK`; (14) Cursor slice-mode prompt dual-list contract pin (#659, pattern parallel to (5a)/(5b)/(5c)/(10)) — a single `SKILL.md` line carries `cursor-wrap-prompt.sh` (anchors to the Cursor slice-mode prompt line — `cursor agent` itself sits on a Bash backslash-continuation predecessor line, so `cursor-wrap-prompt.sh` is the unique on-prompt-line anchor), `slice-files.txt` (anchors to slice mode), the OOS-marking sentence `Mark any finding about a file NOT in slice-files.txt as OOS`, AND the canonical section header literals `### In-Scope Findings` and `### Out-of-Scope Observations` together; pipeline-threaded so a future edit that splits the directive across lines fails closed, and inherently avoids the vacuous-pass risk of a `while read` loop without a non-empty guard; (15) ALL slice-mode external-reviewer prompts carry the dual-list contract (#659) — every `SKILL.md` line containing the OOS-marking sentence `Mark any finding about a file NOT in slice-files.txt as OOS` also contains both `### In-Scope Findings` and `### Out-of-Scope Observations`; non-emptiness guard plus exact-count check (exactly 2 such lines: one Cursor slice prompt + one Codex slice prompt) matches assertion (6)'s pattern (pre-loop `[[ -n ... ]]` plus a `while read` loop with per-line check). The count pin catches accidental addition or removal of one of the slice prompts; the per-line check catches the Codex slice prompt that assertion (14) cannot anchor by tool name (Codex line 190 is a bare double-quoted positional argument to `codex exec` with no tool-specific literal on the line itself); (16) Step 3a description-mode dual-list parsing pin (#659) — a single `SKILL.md` line carries `In description mode`, `dual-list output`, `### In-Scope Findings`, AND `### Out-of-Scope Observations` together (pinning the parser-side mode-conditional wording in Step 3a item 2); (17) Step 3a diff-mode single-list preservation pin (#659) — a single `SKILL.md` line carries `In diff mode`, `single-list output`, AND `entire output` together (blocking a future blanket rewrite from flattening the slice/diff modes back to a single contract). Together (14)/(15)/(16)/(17) anchor the contract introduced by PR #659 so a future edit cannot regress the slice/diff mode split silently. Intentional overlap: assertions (7) and (8) duplicate per-skill coverage already provided by `scripts/test-anti-halt-banners.sh` (whose `ORCHESTRATORS` array includes `skills/review/SKILL.md` and whose `BANNER_SIGNATURE` / `MICRO_SIGNATURE` pin the same substrings). The duplication is intentional for single-file fail locality — accepted duplication, matching the `test-implement-structure.sh` precedent which similarly overlaps with `test-implement-rebase-macro.sh`. Wired into `make lint` via the `test-review-structure` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to `skills/review/SKILL.md` (MANDATORY pointers including the Step-3-entry and round-1-branch callsites, the `Do NOT load` rounds-2+ guard for `voting.md`, the Cursor/Codex quick-review prompt lines carrying the focus-area enum, the anti-halt banner text, the `Continue after child returns` micro-reminder text, the two-mode activation directive carrying `--diff` and `positional description`, the verbatim `--diff cannot be combined with a description ...` abort message, the verbatim `/review requires either --diff ...` no-args error message, the Step 3a substantive-validation collector invocation carrying `--substantive-validation` and `--validation-mode`, the Cursor/Codex description-mode dual-list prompt lines carrying both the OOS-marking sentence and the `### In-Scope Findings` / `### Out-of-Scope Observations` section headers, the Step 3a `In description mode` / `dual-list output` mode-conditional parsing wording, and the Step 3a `In diff mode` / `single-list output` / `entire output` preservation wording) or to any `skills/review/references/*.md` (renaming, removing, altering the opening `**Consumer**:` / `**Binding convention**:` header lines, or adding a new reference without a MANDATORY pointer in `SKILL.md`) must stay in sync with this harness's literal assertions.
