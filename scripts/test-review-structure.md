# scripts/test-review-structure.sh — contract

`scripts/test-review-structure.sh` is the structural regression guard for the `/review` skill's progressive-disclosure topology (closes #306, hardened in #318; three-way slice-activation pins added per #637; substantive-validation flag pin added per #661). Thirteen assertions: (1) `skills/review/SKILL.md` and `skills/review/references/` exist; (2) baseline expected reference files `skills/review/references/domain-rules.md` and `skills/review/references/voting.md` exist; (3) every `skills/review/references/*.md` file on disk is named on at least one `MANDATORY — READ ENTIRE FILE` line in `SKILL.md` using the `references/<basename>` path-token form followed by end-of-line or a non-filename character outside `[A-Za-z0-9._-]` (bidirectional orphan detection — filesystem enumeration is authoritative so a newly added reference that lacks a MANDATORY pointer fails closed; the path-token boundary prevents both a name-containing-name false-pass like `my-voting.md` satisfying the check for `voting.md` AND a suffix/extension false-pass like `foo.md.bak` satisfying the check for `foo.md`); (4) each baseline expected reference (`domain-rules.md`, `voting.md`) appears on at least one MANDATORY line — logically implied by (3) once the filesystem matches the baseline, but kept as a distinct check for clearer diagnostics when the baseline pair specifically regresses; (5) line-scoped callsite pins (#318, pattern parallel to `test-research-structure.sh`'s reciprocal Do-NOT-load pins) — (5a) a single `SKILL.md` line carries `MANDATORY — READ ENTIRE FILE`, `Step 3` (with a non-word-char boundary so `Step 3a` / `Step 30` / `Step 3f` do NOT false-pass), and `references/domain-rules.md` together (pinning `domain-rules.md` to the Step 3 entry callsite); (5b) a single line carries `MANDATORY — READ ENTIRE FILE`, `round 1` (case-insensitive — matches `In round 1` too — with a non-word-char boundary so `round 10` / `round 11` do NOT false-pass), and `references/voting.md` together (pinning `voting.md` to the round-1 branch callsite); (5c) a single line carries `Do NOT load` and `references/voting.md` together (reciprocal rounds-2+ guard). Line-scoped by construction via a grep pipeline that threads each token through its own filter stage while preserving line granularity; a future edit that splits a directive across lines (e.g., moving `voting.md`'s MANDATORY to Step 3 entry or dropping the Do-NOT-load guard) fails closed. Reference-path boundary character outside `[A-Za-z0-9._-]` mirrors (3)/(4); (6) CI-parity focus-area enum check — at least one line in `SKILL.md` contains the literal `code-quality / risk-integration / correctness / architecture` AND every such line also contains `security` on the same line (fails on any enum line missing `security`, mirroring the `agent-sync` job's `UNQUOTED_FILES` loop in `.github/workflows/ci.yaml` — the loop referenced by name rather than line number so the harness does not rot when the workflow changes; per-line enforcement prevents a false-pass when the five tokens appear separately in unrelated prose while the Cursor/Codex quick-review prompt strings regress; same approach as `test-implement-structure.sh` assertion 6); (7) `SKILL.md` contains the anti-halt banner substring `**Anti-halt continuation reminder.**`; (8) `SKILL.md` contains the micro-reminder substring `Continue after child returns` (narrow token without bold wrappers or trailing period, matching `scripts/test-anti-halt-banners.sh`'s `MICRO_SIGNATURE` so a future loop-internal variant like `**Continue after child returns (loop-internal).**` still matches — the canonical contract in `skills/shared/subskill-invocation.md` is this narrower token); (9) each `skills/review/references/*.md` opens with `**Consumer**:` AND `**Binding convention**:` in the first 20 lines (the `/review` native 2-line header schema — deliberately NOT the `/implement` `Consumer`/`Contract`/`When to load` triplet because `/review` references use a different header schema; `head -n 20` gate matches `test-research-structure.sh` and `test-design-structure.sh`, so a future edit that buries the headers mid-file fails closed); (10) three-way slice-mode activation pin (#637, pattern parallel to (5a)/(5b)/(5c)) — at least one `SKILL.md` line carries `Slice mode`, `--slice` (substring match — also satisfied by `--slice-file` since the flag check is reinforced by the dedicated `--slice-file` filter stage), `--slice-file`, AND `positional` (case-insensitive on `positional` since prose/heading variants like `Positional` may carry the contract) together; pipeline-threaded so a future edit that splits the activation directive across lines fails closed; the deliberately permissive substring grep on `--slice` (no trailing space/boundary) lets the contract sit on the canonical activation sentence (`**Slice mode** is active when --slice is set...`), the Flags-bullet enumeration (`(a) --slice <text>, (b) --slice-file <path>, or (c)...`), or the Step 1 subheading (`### Slice mode (--slice, --slice-file, or trailing positional text after --create-issues)`); the harness PASSES whenever at least one such line still exists in `SKILL.md` and FAILS only when no single line carries all four tokens together (e.g., the activation prose was split across lines, or every contract-bearing line lost one of the tokens); (11) verbatim empty-positional abort message pin (#637) — `SKILL.md` contains the literal string `**⚠ --create-issues requires a slice description (--slice <text>, --slice-file <path>, or trailing positional text). Aborting.**` (`grep -Fq` so any wording drift fails closed; the abort message is a user-facing contract that downstream tooling and operator runbooks may depend on); (12) verbatim positional-vs-slice-flag mutual-exclusion abort message pin (#637) — `SKILL.md` contains the literal string `**⚠ Positional slice text cannot be combined with --slice or --slice-file. Aborting.**` (`grep -Fq` same rationale as (11)). Together (10)/(11)/(12) anchor the contracts introduced by PR #638 so a future edit cannot regress them silently. (13) substantive-validation flag pin (#661, pattern parallel to (10)) — at least one `SKILL.md` line carries `collect-reviewer-results.sh`, `--timeout 1860`, `--substantive-validation`, AND `--validation-mode` together; line-scoped via grep pipeline so a future edit that splits the Step 3a collector invocation across lines or drops either flag fails closed under `set -o pipefail`. Anchors the issue #661 contract that banner-only reviewer output (e.g., `Authentication required`) is rejected as `STATUS=NOT_SUBSTANTIVE` rather than passing as `STATUS=OK`. Intentional overlap: assertions (7) and (8) duplicate per-skill coverage already provided by `scripts/test-anti-halt-banners.sh` (whose `ORCHESTRATORS` array includes `skills/review/SKILL.md` and whose `BANNER_SIGNATURE` / `MICRO_SIGNATURE` pin the same substrings). The duplication is intentional for single-file fail locality — accepted duplication, matching the `test-implement-structure.sh` precedent which similarly overlaps with `test-implement-rebase-macro.sh`. Wired into `make lint` via the `test-review-structure` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to `skills/review/SKILL.md` (MANDATORY pointers including the Step-3-entry and round-1-branch callsites, the `Do NOT load` rounds-2+ guard for `voting.md`, the Cursor/Codex quick-review prompt lines carrying the focus-area enum, the anti-halt banner text, the `Continue after child returns` micro-reminder text, the three-way slice-mode activation sentence/enumeration, the verbatim `--create-issues requires a slice description ...` abort message, the verbatim `Positional slice text cannot be combined with --slice or --slice-file ...` abort message) or to any `skills/review/references/*.md` (renaming, removing, altering the opening `**Consumer**:` / `**Binding convention**:` header lines, or adding a new reference without a MANDATORY pointer in `SKILL.md`) must stay in sync with this harness's literal assertions.
