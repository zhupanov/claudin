# scripts/check-bump-version.sh — contract

`scripts/check-bump-version.sh` is the `/implement` Step 8 bump-workflow probe. Two modes: `--mode pre` emits `HAS_BUMP=true|false`, `COMMITS_BEFORE=<N>`, `STATUS=ok|missing_main_ref|git_error`; `--mode post --before-count <N>` emits `VERIFIED=true|false`, `COMMITS_AFTER=<N>`, `EXPECTED=<N>`, `STATUS=ok|missing_main_ref|git_error`. The `STATUS=` line plumbs the `COUNT_COMMITS_STATUS_FILE` side channel from `lib-count-commits.sh` so callers can distinguish a transient git error (corrupted pack, shallow-clone object boundary, PATH shim, etc.) from a legitimate "0 commits ahead" result (closes #172). `--mode post` forces `VERIFIED=false` whenever `STATUS != ok` — the numeric commit-delta is never trusted under degraded git state — which is the fail-closed invariant that fixes the historical silent-zero false-pass bug. Any unknown or empty token received from the side channel is normalized to `STATUS=git_error` (mirrors `verify-skill-called.sh`'s default branch that maps unknown to `REASON=git_error`). `scripts/test-check-bump-version.sh` is its regression harness covering all three status paths × both modes plus the `origin/main`-only fallback and the #172 fail-closed regression guard, wired into `make lint` via the `test-check-bump-version` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint does not follow Makefile-only references. Edits to the script must stay in sync with the harness.
