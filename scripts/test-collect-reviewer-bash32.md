# scripts/test-collect-reviewer-bash32.sh — contract

`scripts/test-collect-reviewer-bash32.sh` is the regression harness for the bash 3.2 portability hazard at `scripts/collect-reviewer-results.sh:405` (issue #511). Three cases: (1) static grep that pins the safe-expansion idiom `"${VAL_ARGS[@]+"${VAL_ARGS[@]}"}"` at the validator call site — runs on every bash version and is the Linux-CI regression backstop because bash 4.4+ no longer exhibits the empty-array nounset hazard at runtime; (2) dynamic empty-`VAL_ARGS` exercise of the actual collector under `/bin/bash` 3.x (≥200-word substantive fixture, pre-created `.done` sentinel containing `0`, basename containing `cursor` so `derive_tool` attributes correctly) asserting `STATUS=OK` AND clean stderr — skip-with-loud-message on bash 4+; (3) dynamic `--validation-mode` forwarding pin under `/bin/bash` 3.x using a literal `NO_ISSUES_FOUND` fixture with both a positive assertion (`--validation-mode` present → `STATUS=OK`) and a negative control (`--validation-mode` absent → `STATUS=NOT_SUBSTANTIVE`) — skip-with-loud-message on bash 4+. Wired into `make lint` via the `test-collect-reviewer-bash32` target and the `test-harnesses` aggregator. The test script is added to `agent-lint.toml`'s exclude array because it is a Makefile-only harness (referenced only from `Makefile`, not from any SKILL.md or other runtime path) — same shape as `scripts/test-validate-research-output.sh`. Edits to the substantive-validation block in `scripts/collect-reviewer-results.sh` (around lines 372-412) MUST keep the safe-expansion idiom at the validator call site or Case 1 will fail; if the call-site shape changes (e.g., introducing a helper function or splitting the loop), update the Case 1 grep regex AND the `scripts/collect-reviewer-results.md` invariant in the same PR.
