# scripts/deny-edit-write.sh — contract

`scripts/deny-edit-write.sh` is the skill-scoped PreToolUse hook registered in `skills/research/SKILL.md`'s `hooks:` frontmatter for matcher `Edit|Write|NotebookEdit` — it permits the call only when the tool's target path resolves to an absolute path under canonical `/tmp`, and denies every other outcome (missing path, relative path, traversal outside `/tmp`, symlink cycle, canonicalization failure, `jq` runtime failure). The script reads stdin JSON and extracts the target via `jq -r '[.tool_input.file_path, .tool_input.notebook_path] | map(select(type == "string" and length > 0)) | .[0] // empty'` — `map(select(length > 0))` is required because `jq`'s `//` treats the empty string as present, so a naïve `// empty` would let an empty `file_path` shadow a valid `notebook_path`. NotebookEdit uses `notebook_path`; the length-aware fallback prevents fail-open on that shape. Path resolution mirrors `scripts/block-submodule-edit.sh`: bounded symlink walk (max depth 40; cycle → deny), nearest-existing-ancestor probe via `dirname`, then canonicalization via `cd … && pwd -P` to handle macOS `/tmp` → `/private/tmp` aliasing. The allowed root is pre-computed once as `$(cd /tmp && pwd -P)`; the decision is exact equality or `$ALLOWED_ROOT/` prefix. On allow the script emits empty stdout and exits 0; on deny it emits a fixed-literal `hookSpecificOutput` JSON envelope and exits 0. The deny envelope is composed only from fixed ASCII literals (single fixed reason string, no runtime interpolation into the deny JSON) and is byte-identical across three branches: the top-level `jq`-absent static `printf` path, the `block()` helper's `jq -cn` path, and `block()`'s inner `jq -cn || printf '<static>'` fallback (which protects against a `jq` runtime failure degrading to exit 0 with empty stdout = silent-allow). This mirrors the `jq || printf` precedent in `scripts/block-submodule-edit.sh:25-32`. The hook is the sole mechanical enforcer of the `/tmp`-only policy; `allowed-tools` in `skills/research/SKILL.md` declares the orchestrator's surface (including `Edit`, `Write`, `NotebookEdit`, and `Skill`) but does NOT confine writes to `/tmp` — see `SECURITY.md` for the full residual-risk framing (hook `permissionDecision` semantics may vary by Claude Code version; `Skill` is deliberately unscoped). The hook matcher MUST remain `Edit|Write|NotebookEdit` — adding `Skill` would deny `/issue` delegations, breaking the read-only-repo contract's child-invocation story. `scripts/test-deny-edit-write.sh` is the table-driven regression harness (repo-deny, `/tmp`-allow for both new and existing files, `/tmp/../etc/passwd`-deny, relative-deny, NotebookEdit `notebook_path` allow/deny, empty-`file_path`-with-valid-`notebook_path` allow (guards the jq `//` empty-string edge case), empty-path fail-closed deny, malformed-JSON deny, idempotency, `jq`-absent static fallback byte-identity), wired into `make lint` via the `test-deny-edit-write` target; the harness FAILs (exit 1) when its own `jq` is missing because the assertions need a JSON parser. Edits to the hook must stay in sync with the harness. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references.
