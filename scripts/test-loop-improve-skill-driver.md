# scripts/test-loop-improve-skill-driver.sh — contract

`scripts/test-loop-improve-skill-driver.sh` is the structural + behavioral regression harness for `skills/loop-improve-skill/scripts/driver.sh` — the Option B bash-driver topology for `/loop-improve-skill` introduced by umbrella #273 and since refactored so the per-iteration body (judge → design → im) lives in the shared kernel `skills/improve-skill/scripts/iteration.sh` (invoked via direct bash call from the driver's loop body; no `claude -p` nesting). Halt class stays eliminated by construction: iteration.sh runs each child skill as a fresh `claude -p` subprocess and the driver → iteration.sh edge is a plain bash call (no model turn between return and post-call bash).

**Structural asserts** on `driver.sh` source contents: loop-driver contracts (`parse-skill-judge-grade.sh` for final re-judge, `claude --version`, `set -euo pipefail`, LOOP_TMPDIR `/tmp/` + `/private/tmp/` prefix-guard literals, `'..'` rejection check, `redact-secrets.sh` in the close-out gh-comment pipeline, `cleanup-tmpdir.sh`, the `max iterations (10) reached` / `grade A achieved on all dimensions` / `grade A achieved after final post-iter-cap` exit-reason literals, `Grade History` / `Infeasibility Justification` headings, `session-setup.sh`, `gh issue create` + `gh issue comment`, `LOOP_IMPROVE_SKIP_PREFLIGHT` + `--skip-preflight`). Iteration delegation tokens: `LARCH_ITERATION_SCRIPT_OVERRIDE` env var (test-only redirect), `skills/improve-skill/scripts/iteration.sh` default kernel path, `ITERATION_SCRIPT` variable binding, and the four flags passed through to iteration.sh (`--work-dir`, `--iter-num`, `--breadcrumb-prefix`, `--issue`). KV-footer parse tokens: `### iteration-result` delimiter + `ITER_STATUS` / `EXIT_REASON` / `PARSE_STATUS` / `GRADE_A` / `NON_A_DIMS` keys. Retained slim `invoke_claude_p` (used only for the Step 5a post-iter-cap re-judge) must preserve FINDING_7 (`--plugin-dir` + `"$CLAUDE_PLUGIN_ROOT"`), FINDING_9 (`< "$prompt_file"` STDIN redirect), FINDING_10 (`2> "$stderr_file"` + `local stderr_file="${out_file}.stderr"` sidecar), and the non-interactive permission contract `--permission-mode bypassPermissions` (issue #614, sibling to #585) so an in-child tool-permission prompt cannot stall the subprocess until the 1200s watchdog fires. The per-iteration FINDING_9/10 invariants for `/skill-judge` / `/design` / `/im` now live in `iteration.sh` and are pinned by `scripts/test-improve-skill-iteration.sh` — intentionally no longer asserted here.

**Behavioral fixtures** (best-effort, stubbed `claude` + `gh` on PATH, plus a stub iteration shim redirected via `LARCH_ITERATION_SCRIPT_OVERRIDE`): grade-A short-circuit at iter 1, no_plan infeasibility exit, im_verification_failed infeasibility exit. The stub shim emits deterministic KV footers so the harness exercises the driver's parse + break logic directly rather than re-coupling to the real iteration kernel.

Wired into `make lint` via the `test-loop-improve-skill-driver` target; listed in `agent-lint.toml` exclude (Makefile-only reference). Edits to `driver.sh` must stay in sync with the structural assertions in this harness — when a contract token literal changes, update the harness in the same PR. Edits to `iteration.sh` that change the KV footer key set MUST update both this harness's driver-side parse assertions AND `scripts/test-improve-skill-iteration.sh`'s kernel-side emission assertions. The prior `/loop-improve-skill-iter` inner skill and its continuation harness have been retired.
