# skills/compress-skill/scripts/build-feature-description.sh — contract

`skills/compress-skill/scripts/build-feature-description.sh` resolves the target skill, enumerates the transitively-reachable `.md` set inside the skill directory, snapshots baseline byte/line sizes for each file, and composes the `/implement` feature description for `/compress-skill`. Accepts a single positional argument `<skill-name-or-absolute-path>`: a bare skill name (`^[a-z][a-z0-9-]*$`, no colons; a single leading `/` is tolerated and stripped) is resolved via the probe order `${CLAUDE_PLUGIN_ROOT}/skills/<name>/` → `$PWD/skills/<name>/` → `$PWD/.claude/skills/<name>/` → `${CLAUDE_PLUGIN_ROOT}/.claude/skills/<name>/`; an absolute path with more than one segment is used as-is. Transitive `.md` discovery is delegated to the sibling `discover-md-set.sh` (wrapping `discover-md-set.py`); discover stdout is captured for the `FILE_COUNT=` parse while stderr streams through so warnings stay visible and cannot contaminate the parse. Stdout contract (always one `KEY=VALUE` per line) on success: `STATUS=ok`, `TARGET_DIR=<absolute-path-to-skill-dir>`, `SKILL_NAME=<basename>`, `FILE_COUNT=<n>`, `FEATURE_FILE=<absolute-path-to-temp-file-containing-feature-description>` (the temp file holds the self-contained feature description — scope, file list, baseline-sizes Markdown table, Strunk & White style guide, anti-patterns, per-file judgment rules, and PR-body "Token budget" requirement; the **caller owns `FEATURE_FILE`'s lifetime** — the script emits the path and exits without cleanup so the caller can read the contents, then the caller MUST `rm -f "$FEATURE_FILE"` once the contents are captured, otherwise every `/compress-skill` invocation leaks one temp file under the system temp directory (`mktemp -t`, typically `$TMPDIR` — e.g., `/var/folders/...` on macOS or `/tmp` on Linux)); on failure: `STATUS=not_found|bad_name` with a matching `ERROR=<message>` line. Fail-closed: exit 0 on every `STATUS=` branch so the caller can parse stdout deterministically; exit 1 only on internal errors (unparseable args, `mktemp` failure, `discover-md-set.sh` failure). The script's in-file header (lines 1–37) is the authoritative developer-facing spec for flag list, stdout KV grammar, resolution order, and exit-code semantics — read the header comment first when editing. This sibling `.md` exists so the co-location convention (see `AGENTS.md § Editing rules`) is uniformly applied across `skills/<name>/scripts/` — edits that change the stdout contract, resolution order, or the shape of the composed feature description must update both the in-file header and this sibling in the same PR.
