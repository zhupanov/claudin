# skills/create-skill/scripts/post-scaffold-hints.sh — contract

`skills/create-skill/scripts/post-scaffold-hints.sh` is the post-scaffold reminder emitter invoked by `/create-skill`. In plugin-dev mode it emits: dual-form Skill permission hints (`Skill(<NAME>)` and `Skill(larch:<NAME>)`) + README "Strict-permissions consumers — Skill permission entries" subsection pointer; README Skills catalog + feature matrix row; `.claude/settings.json` permissions.allow entries with `sort -u` instruction; `docs/workflow-lifecycle.md` update (Skill Orchestration Hierarchy OR Delegation Topology subsection, plus Standalone Usage); conditional "when applicable" reminders for `docs/agents.md` (when the new skill spawns subagents), `docs/review-agents.md` (when it alters reviewer composition), and `AGENTS.md` Canonical sources list (when the new skill introduces a shared script or is itself a canonical source). `scripts/test-post-scaffold-hints.sh` is its regression harness, wired into `make lint` via the `test-post-scaffold-hints` target; edits to the hints script must stay in sync with the harness. Contract tokens asserted by the harness: dual Skill permission lines, README-subsection title, `sort -u` instruction, `docs/workflow-lifecycle.md`, `Delegation Topology`, `docs/agents.md`, `docs/review-agents.md`, `AGENTS.md Canonical sources`, and the literal `when applicable` wording — do not silently drop any of these without updating the harness in the same edit.
