# skills/fix-issue/scripts/parse-prose-blockers.sh — contract

`skills/fix-issue/scripts/parse-prose-blockers.sh` is the prose-dependency parser invoked by `prose_open_blockers()` in `skills/fix-issue/scripts/blocker-helpers.sh` (the shared blocker-resolution library sourced by both `find-lock-issue.sh` and `umbrella-handler.sh`). It extracts same-repo issue numbers from a single document's text (issue body, or one comment body at a time) matched against the conservative keyword set (`Depends on`, `Blocked by`, `Blocked on`, `Requires`, `Needs` + whitespace + `#N`). Inline code spans (backtick-delimited) are stripped before any other normalization so dependency keywords inside code like `` `Depends on #99` `` are not false-positive matched; only matched backtick pairs are removed (unmatched trailing backticks are harmless). Fenced code blocks (triple-backtick) are NOT stripped — the mitigation targets inline spans only. Emphasis wrappers (`*`, `_`) are normalized after code-span removal so the motivating `**#150**` formatting is matched; link-target forms (`[#150](url)`) and cross-repo references remain NON-matches by construction. The parser is fail-open (empty stdout + exit 0 on any error), so correctness is guarded by `skills/fix-issue/scripts/test-parse-prose-blockers.sh` — its offline regression harness wired into `make lint` via the `test-parse-prose-blockers` target. The test script is added to `agent-lint.toml`'s exclude list because agent-lint's dead-script rule does not follow Makefile-only references. Edits to the parser must stay in sync with the harness.
